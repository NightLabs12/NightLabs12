## 🎯 My Approach
I believe successful business systems rely on data integrity and understanding the actual workflows before writing any code.

My process:
1. **Understand the Workflow first** — I map out the actual business flow (like procurement approvals or inventory tracking) before designing the system structure.
2. **Prioritize Data Integrity** — Especially in ERP and financial modules, dirty data causes critical errors. I build strict database-level validations, relationships, and audit trails.
3. **Database & Query Performance** — High throughput shouldn't slow down reports. I optimize queries, design efficient indexes, and structure schemas in PostgreSQL/MySQL for speed.
4. **Offline-First Resilience** — For mobile apps used in the field, connection drops are common. I implement local caching (SQLite) to ensure zero data loss during operations.
